# Activité – Choisir une architecture adaptée

## 🎯 Objectif pédagogique

Cette activité a pour but de :

- Analyser un contexte réel
- Identifier l’architecture système la plus adaptée
- Justifier des choix techniques
- Comparer différentes approches (microservices, client-serveur, P2P)
- Comprendre l’impact du contexte sur l’architecture
- Développer l’argumentation technique

---

# 🕒 Déroulement de l’activité

## Étape 1 — Réflexion individuelle (5 minutes)

Chaque apprenti reçoit **un scénario différent**.

Travail individuel, sans internet, avec la présentation ppt :

- Identifier le type d’architecture le plus adapté
- Proposer les composants principaux du système
- Identifier 2 risques techniques
- Donner 1 justification principale (scalabilité, résilience, simplicité, coût…)

⚠️ Aucune discussion à cette étape.

---

## Étape 2 — Mise en commun par groupe de 3 (10 minutes)

Les apprentis se mettent en groupe de 3.

⚠️ Chaque membre du groupe doit avoir un **scénario différent**.

Chaque apprenti :

1. Présente son scénario
2. Explique son architecture proposée
3. Justifie ses choix

Les autres membres :

- Posent des questions
- Challengent les décisions
- Proposent des améliorations

Objectif :

- Améliorer chaque proposition
- Identifier les différences entre les architectures
- Comprendre pourquoi une solution est adaptée à un contexte donné

---

## Étape 3 — Regroupement par scénario (15–20 minutes)

Les apprentis se regroupent avec ceux qui ont **le même scénario**.

### Objectifs

- Comparer leurs propositions initiales  
- Construire une architecture commune améliorée  
- Choisir un **mode de déploiement adapté**
- Préparer une présentation synthétique et la présenter devant la classe

### Travail demandé

Le groupe doit produire :

- Un **schéma d’architecture clair**
- Les **composants clés**
- Les **arguments principaux**
- Les **risques identifiés**
- Le **mode de déploiement choisi** (On-prem, IaaS, PaaS, SaaS ou hybride)
- Pour **chaque composant**, indiquer :
  - Où il est déployé (type de déploiement choisi)
  - S’il peut être déployé automatiquement (CI/CD, IaC, etc.)

Le schéma doit clairement montrer :
- Les flux entre composants
- L’emplacement de déploiement de chaque élément
- Les éléments automatisés


---

# 📌 Livrables attendus

Pour chaque groupe :

- Un schéma d’architecture
- Une justification argumentée
- Les risques principaux identifiés

---

# 🧠 Ce que vous devez être capables d’expliquer

- Pourquoi une architecture est adaptée à un contexte spécifique
- Quand utiliser microservices, P2P ou client-serveur
- L’impact du trafic, de la résilience et de la complexité
- Le lien avec le cloud et les modèles XaaS
- Pourquoi une solution simple est parfois préférable

---

# 🎓 Compétences travaillées

- Analyse d’architecture
- Comparaison de modèles systèmes
- Justification technique
- Communication professionnelle
- Travail collaboratif
